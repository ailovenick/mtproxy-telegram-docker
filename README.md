# MTProxy (mtg v2) с Fake-TLS

MTProto прокси на языке Go с маскировкой трафика под HTTPS.
Официальный репозиторий: https://github.com/9seconds/mtg

## Конфигурация и масштабирование
Проект по умолчанию содержит конфигурацию для трех независимых инстансов прокси (как пример). Количество масштабируется добавлением или удалением блоков в `docker-compose.yml` и созданием соответствующих `.toml` файлов по аналогии.

### Файлы и порты:
- **Контейнер `mtproxy`**: Порт `8081`, конфиг `config.toml`.
- **Контейнер `mtproxy2`**: Порт `8082`, конфиг `config2.toml`.
- **Контейнер `mtproxy3`**: Порт `8083`, конфиг `config3.toml`.

### Структура config.toml:
- `secret`: Секретный ключ (один на один конфиг).
- `bind-to`: Внутренний адрес и порт контейнера (по умолчанию `0.0.0.0:8080`).

## Генерация секрета (секретный ключ)
В `mtg v2` секрет включает префикс протокола, случайный ключ и hex-код домена для маскировки (Fake-TLS). 

**Рекомендация:** Не использовать предустановленные секреты из репозитория. Сгенерированные ключи необходимо вручную вставить в соответствующие `.toml` файлы в поле `secret`.

### Способ 1: С использованием Docker (Официальный инструмент)
Генерация секрета через встроенную утилиту `mtg`:
```bash
docker run --rm nineseconds/mtg:2 generate-secret google.com
```
*Флаг `--hex` возвращает строку в формате HEX (начинается с `ee`). Без флага — в формате Base64.*

### Способ 2: С использованием openssl (Ручная генерация)
Генерация HEX-секрета для домена `google.com`:
```bash
echo "ee$(openssl rand -hex 16)$(echo -n 'google.com' | xxd -p | tr -d '\n')"
```

## Запуск
Запуск всех контейнеров в фоновом режиме:
```bash
docker-compose up -d
```
При внесении изменений в `.toml` файлы требуется перезапуск соответствующего контейнера:
```bash
docker-compose restart <имя_контейнера>
```

## Формирование ссылок для Telegram
Ссылка формируется по шаблону:
`tg://proxy?server=<IP>&port=<PORT>&secret=<SECRET>`

Или (HTTP-вариант):
`https://t.me/proxy?server=<IP>&port=<PORT>&secret=<SECRET>`

### Параметры:
- `<IP>`: Публичный IP-адрес или домен сервера.
- `<PORT>`: Внешний порт из `docker-compose.yml` (8081, 8082 или 8083).
- `<SECRET>`: Соответствующий секретный ключ из файла конфигурации.
